# 用户订阅管理系统 — 技术实现方案

## Context

基于 `需求文档.md` 构建全栈用户订阅服务时长管理系统。仅管理员可登录，管理普通用户的订阅时长、到期时间和连续包月状态。

## 技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 全栈框架 | Next.js 15 (App Router) | API Routes + 纯客户端渲染 |
| UI 组件 | Ant Design 6.x | `ConfigProvider` + 中文 locale，原生 React 19 支持 |
| HTTP 客户端 | Axios | 封装统一 `request`，`withCredentials: true` |
| 数据库 | Supabase (PostgreSQL) | 托管 Postgres + REST API |
| 认证 | Supabase Auth | 邮箱密码登录 + JWT + Cookie 托管 |
| 密码加密 | Supabase 内置 bcrypt | 无需自行实现 |

## 2. 数据库设计

### profiles 表（Supabase）

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK → `auth.users.id` | 关联 Supabase Auth 用户 |
| name | VARCHAR(50) | NOT NULL | 姓名 |
| phone | VARCHAR(20) | NOT NULL, UNIQUE | 手机号 |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'user' | admin / user |
| created_at | TIMESTAMPTZ | DEFAULT now() | — |
| updated_at | TIMESTAMPTZ | DEFAULT now() | — |

### subscriptions 表

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | — |
| user_id | UUID | NOT NULL, UNIQUE → `auth.users.id` | 一对一关联 |
| duration | INTEGER | NOT NULL, DEFAULT 0 | 订阅时长（月） |
| expire_date | DATE | NOT NULL | 到期时间 |
| is_continuous | BOOLEAN | NOT NULL, DEFAULT false | 是否连续包月 |
| created_at | TIMESTAMPTZ | DEFAULT now() | — |
| updated_at | TIMESTAMPTZ | DEFAULT now() | — |

### 用户邮箱映射

Supabase Auth 需要合法邮箱格式。用户按手机号登录，系统内部映射为 `{phone}@user.local`：

```
用户输入：13800000000 + 123456
系统转换：13800000000@user.local + 123456 → signInWithPassword()
```

用户完全无感，登录页只展示手机号输入框。

## 3. API 设计

统一响应格式：`{ code: 0, message: "ok", data: null }`

### 认证接口 `/api/auth`

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| POST | /api/auth/login | 否 | `{ phone, password }` → `{phone}@user.local` 登录，校验 admin 角色 |
| POST | /api/auth/logout | 否 | `supabase.auth.signOut()` |
| GET | /api/auth/me | 是 | `supabase.auth.getUser()` + 联查 profiles |

### 用户接口 `/api/users`（全部需要 admin 鉴权）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/users | 查询 `role = 'user'` 的用户及 subscriptions |
| PUT | /api/users/[id]/subscription | upsert 订阅（`onConflict: user_id`） |

### 参数校验

| 接口 | 规则 |
|------|------|
| login | phone 必填、合法手机号；password 必填、≥6 位 |
| update subscription | duration ≥0 整数；expireDate 必填、合法日期 |

## 4. 认证流程

```
登录：
用户输入手机号+密码
  → API: phone + '@user.local' → supabase.auth.signInWithPassword()
  → 查询 profiles 表检查 role === 'admin'
  → 非 admin → signOut() + 403
  → admin → session cookie 自动由 @supabase/ssr 管理

请求鉴权：
浏览器自动携带 cookie
  → middleware.ts: updateSession() 刷新 session
  → API Route: verifyAuth() → supabase.auth.getUser()
```

**关键点**：
- JWT 签发/验证、Cookie 管理全部由 Supabase 内置处理
- 项目不自行管理 token，只调用 `getUser()` 获取当前用户
- `service_role` key 仅用于 seed 脚本（`auth.admin.createUser`），不暴露给前端

## 5. 前端组件树与数据流

```
<RootLayout>                              # layout.tsx — 服务端组件
  <AppProvider>                           # 'use client' — ConfigProvider + AuthGuard
    /login  → <LoginPage>                 # Card + Form（phone + password）
                └─ 登录成功 → router.push('/')
    /       → <UserManagementPage>        # 状态持有者：users[], loading, modalOpen, selectedUser
                ├── <UserTable>           # loading/empty，编辑按钮回调
                └── <EditSubscriptionModal>
                       └─ Form: duration + expireDate(DatePicker) + isContinuous(Switch)
```

### 编辑流程

```
点击「编辑」→ setSelectedUser + setModalOpen(true)
  → Modal useEffect + form.setFieldsValue 预填
  → 用户修改 → 确定
  → handleSave: userService.updateSubscription() → 刷新列表 → 关闭 Modal + message.success
```

## 6. 认证与角色控制

| 场景 | 结果 |
|------|------|
| admin 用户登录 | ✅ 成功，看到普通用户订阅列表，可编辑 |
| 普通用户登录 | ❌ "无权限登录管理平台" |
| 未登录访问 `/` | AuthGuard 检测 → 跳转 `/login` |
| API 无 cookie | `verifyAuth()` 返回 null → 401 |
